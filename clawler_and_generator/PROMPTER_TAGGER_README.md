# 题目进一步分类打标签程序 (prompter_tagger.py)

## 程序简介

本程序使用DeepSeek大语言模型对已经初步分类的Codeforces题目进行更细粒度的自动分类和打标签。程序会递归地分析每个文件夹内的题目，判断是否需要进一步细分，并自动创建子分类文件夹。

## 功能特点

1. **智能判断是否需要细分**：通过分析题目内容和题目数量，自动判断当前分类是否足够精细（少于30道题目通常无需细分）
2. **自动生成子分类标签**：基于题目的算法知识点和应用场景，自动建议合适的子分类
3. **多次抽样优化**：通过多次随机抽样获取子分类建议，确保分类的准确性
4. **智能合并相似标签**：自动识别并合并意义相同的分类标签
5. **并发处理**：使用线程池并发处理题目分类，并发数为100，大幅提升处理速度
6. **断点续传**：支持中断后继续运行，自动跳过已处理的文件夹
7. **自动分类和移动文件**：将每道题目自动分类到最合适的子分类文件夹中
8. **递归处理**：自动递归处理所有子文件夹，直到所有分类都足够精细
9. **生成层级结构图**：最终生成完整的题目分类层级结构图
10. **进度跟踪**：实时统计成功和失败数量，显示总耗时

## 核心流程

1. **获取题目数量**：统计当前文件夹内的题目数量N
2. **判断是否需要细分**：随机抽取45个题目，结合当前tag和题目数量，调用LLM判断是否需要进一步划分
3. **获取子分类建议**：如果需要细分，多次（log₂N次）随机抽样获取子分类建议
4. **合并相似标签**：将多次获取的标签取并集，并调用LLM合并意义相同的标签
5. **创建子文件夹**：为每个子分类创建对应的文件夹
6. **分类和移动**：对每个题目进行分类，更新JSON文件并移动到对应子文件夹
7. **递归处理**：进入每个子文件夹继续上述流程，直到无需再细分
8. **生成层级图**：生成完整的文件夹层级结构图

## 输出文件

- **JSON文件更新**：每个题目的JSON文件会添加新的`layerN_tag`字段
- **文件夹结构**：自动创建子分类文件夹并移动文件
- **层级结构图**：在`tidy_problem_set`文件夹下生成`hierarchy_tree.txt`文件

## Token优化策略

为了在保证质量的同时节省token消耗：

1. **题目摘要**：只提取关键信息（ID、名称、难度、标签、描述前200字符、教程前150字符、代码示例前100字符）
2. **抽样展示**：在判断是否需要划分时，只展示前10个题目的信息
3. **控制token数**：每个API调用设置合理的max_tokens限制
4. **精确prompt**：使用简洁明确的prompt，避免冗余信息

## 注意事项

1. **API调用频率**：程序会进行大量的API调用，请注意API配额
2. **运行时间**：根据题目数量和分类层级，程序可能需要较长时间运行（并发处理可大幅提升速度）
3. **网络连接**：需要稳定的网络连接以调用DeepSeek API
4. **文件备份**：建议在运行前备份`tidy_problem_set`文件夹
5. **中断恢复**：程序支持断点续传！如果中断，重新运行程序会自动跳过已处理的文件夹
6. **并发控制**：默认并发数为100，可以根据机器性能调整`MAX_CONCURRENT`参数
7. **进度文件**：程序会在`tidy_problem_set`文件夹下生成`classification_progress.json`进度文件
8. **文件夹名称**：LLM生成的标签中可能包含特殊字符，程序会自动清理为Windows允许的字符

## 故障排除

### API调用失败

- 检查API密钥是否正确
- 检查网络连接是否正常
- 检查DeepSeek API服务是否可用

### 编码问题

Windows控制台可能出现中文乱码，但不影响程序功能。如需查看正确中文，可以：
- 使用支持UTF-8的终端（如Git Bash）
- 或者查看生成的`hierarchy_tree.txt`文件

### 文件权限问题

确保程序对`tidy_problem_set`文件夹有读写权限。

## 技术支持

如有问题，请检查：
1. Python环境和依赖包是否正确安装
